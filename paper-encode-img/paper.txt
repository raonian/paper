# 选题的背景和意义（600-800字）
## 所选课题的历史背景、国内外研究现状和发展趋势。（1）国内外研究现状，讲述本课题目前在国内外研究状况，介绍各种观点及异同、存在的争论焦点等，同时阐述自己的观点；（2）发展趋势部分，表述本课题目前国内外研究已经达到的水平，存在的问题以及发展趋势等，指明研究方向，提出可以解决的方法。

```
当今时代，随着互联网的发展，无论是手机还是电脑，数字图像在人们的工作、学习以及交流过程中，一直发挥着最基本、最重要的作用。图像比文字含有更丰富的信息，给人更直观的感受，由此，带来的问题就是：图像内容越丰富，画质越清晰，需要的存储空间也越多，对于磁盘空间和网络传输的要求就越高。随着如今的电子设备的硬件配置与日俱增，屏幕分辨率也越来越高，对于图像的清晰度的需求也更强烈。
由于原始图像的庞大数据量无法满足上述需求，因此，为了兼顾图像清晰度和减少体积，在数字图像领域，压缩技术一直备受关注。从数字化电视信号的发明和应用开始，数十年的发展历程中，涌现出了众多具有革命性意义的图像压缩技术和方法，包括小波变换等技术。
人们结合根据不同的使用场景开发了多种图像编码格式，用于压缩原始图像的尺寸，比如：有损压缩jpeg、无损压缩png、比较新的webp等等。这些编码格式都由多种不同的压缩算法组合而成，以目前最流行的jpeg格式为例，其使用了离散余弦变换(DCT)、量化、行程编码(RLE)、熵编码等多个步骤，上一个步骤的输出作为下一步的输入，这样可以充分利用各个压缩算法特性，达到理想的压缩效果。（本文中“压缩”和“编码”意义相同，如未作特别说明，通常能够互换）。其压缩率可以达到10:1到40:1，大大降低了存储和传输图片的磁盘和网络要求。
然而上述编码过程相当复杂，计算量大，程序执行性能较低，过程繁琐，数值精度，并且量化过程还会造成原始数据丢失等等问题，对于行程编码也并不能够创造足够好的条件。
本文的目的在于提出一种算法组合，使整体压缩计算过程相对简单，图像原始数据不损失，并且能够充分利用行程编码(RLE)的优势。同时希望基于这种思想来探索更多压缩算法组合的可能性。

```





# 研究的基本内容和拟解决的主要问题（500-700字）
## 包括课题的提纲和方案论证，研究的主要目的、论点、论据等，以及要达到的结论。以上内容给予粗略的但必须是清楚的介绍。

```
设计思路主要围绕降低压缩过程复杂度、保持数据完整性的方向展开。叙述过程中会插入实际程序代码，来展示压缩过程、程序复杂度。同时附上程序运行的结果，来展示程序执行效率和最终产生的效果。
整体的论证步骤如下：

1、准备原始图像数据，要求图像大小分别为 50KB 以内、500KB 左右、1MB 以上，图像格式 bmp、jpeg 各一份，共 3 组 6 张图片。

2、分别对每组中 bmp 格式图像数据按其分辨率 n * m 划分为 n 行，m 列。然后使用 BWT (Burrows-Wheeler Transform) 算法对每行数据进行处理。处理后，每行数据将比之前多 1 个元素，即 n + 1。并且相同数据元素会聚集在一起，这个特性可以对后面的压缩过程创造良好的条件。

3、对以上结果数据进行相邻元素差量运算，产生大量的 0 值元素。

4、对以上结果数据进行行程编码 (RLE)。

5、压缩完毕，将每组结果文件大小与原始 bmp 格式对比，再与 jpeg 格式对比。

6、解码，逆过程还原图像数据, 最后与原始数据比较有无数据丢失。

以 BWT 作为核心算法，是因为其可以将相同元素聚集在一起，差量运算之后，非常有利于行程编码。同时还能将数据还原，而不损失数据，付出的代价仅仅是每行多出一个元素用来记录相应位置。另外，其程序实现的复杂度也相对较低。

```
